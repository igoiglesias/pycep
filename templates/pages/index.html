Por que usar o PyCEP?

⚡

Cache inteligente

Duas camadas: cache em memória (TTL de 1h) para respostas instantâneas e SQLite para persistência. CEPs já consultados nunca tocam APIs externas.

🔄

Fallback automático

Se o ViaCEP falhar, a BrasilAPI responde. Seus usuários nunca veem erro por indisponibilidade de terceiros.

🔗

Um GET e pronto

Sem SDK, sem API key para testes, sem configuração. Uma rota REST que retorna JSON — funciona com qualquer linguagem.

Documentação

GET /cep/{cep}

Retorna o endereço completo em JSON. Aceita CEP com ou sem hífen.

curl http://localhost:8000/cep/01001000

Resposta:

{
  "content": {
    "cep": "01001-000",
    "logradouro": "Praça da Sé",
    "bairro": "Sé",
    "localidade": "São Paulo",
    "uf": "SP"
  }
}

Integração rápida

JavaScript:

const r = await fetch("/cep/01001000");
const { content } = await r.json();
console.log(content.logradouro);

Python:

import httpx

r = httpx.get("http://localhost:8000/cep/01001000")
print(r.json()["content"]["logradouro"])

Como funciona o cache:

  1. Cache em memória (1h TTL) — resposta imediata
  2. SQLite — persistente entre reinícios
  3. APIs externas — ViaCEP → BrasilAPI (fallback)
  4. Fila de atualização — dados com 30+ dias são renovados em background

Perguntas Frequentes

O PyCEP é gratuito?

Sim. O PyCEP é open source e gratuito para uso pessoal e comercial.

De onde vêm os dados?

Consultamos o ViaCEP como fonte primária e a BrasilAPI como fallback. Os dados são cacheados localmente em memória e em SQLite.

Os dados ficam desatualizados?

Não. CEPs com mais de 30 dias entram em uma fila de atualização automática que renova os dados em background, sem impactar a performance.

Preciso de conta para usar a API?

Para testar aqui na página, não. Para uso contínuo via API, crie uma conta gratuita para acesso autenticado.

Posso contribuir com o projeto?

Claro! O projeto é desenvolvido ao vivo no YouTube e o código é aberto. Contribuições são bem-vindas.
